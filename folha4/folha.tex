\item A função \texttt{foldl :: (a -> b -> a) -> a -> [b] -> a}  é tal que
(\texttt{foldl f z xs}) devolve o resultado da aplicação sucessiva de f usando como primeiro argumento da função f, o valor z na primeira a aplicação e o resultado da aplicação anterior nas outras aplicações; como segundo argumento de f, os valores
da lista xs a partir da esquerda.  Por exemplo,

\begin{lstlisting}[language=Haskell]
foldl (/) 64 [4,2,4]  == ( ( ( 64 / 4 ) / 2 ) / 4 ) == 2.0
\end{lstlisting}

A definição recursiva de foldl é:

\begin{lstlisting}[language=Haskell]
foldl f z [ ] = z
foldl f z (x : xs) = foldl f (f z x ) xs
\end{lstlisting}

A função \texttt{scanEsq} guarda o resultado das aplicações sucessivas realizadas por \texttt{foldl}.

\begin{enumerate}

\item Defina função \texttt{scanEsq :: (b -> a -> b) -> b -> [a] -> [b]}
tal que (\texttt{scanEsq f z xs}) é uma lista
de sucessivas aplicações da função $f$ com os valores da
lista xs a partir da esquerda. Por exemplo,

\texttt{scanEsq 64 (/) [4,2,4] = [64.0, 16.0, 8.0, 2.0]}

\end{enumerate}

\item O fecho de Kleene é uma operação unária aplicada a conjuntos. A aplicação do fecho de Kleene num conjunto A é escrito como
$A^*$. Se $A$ é um alfabeto de uma linguagem, então $A^*$ é o menor superconjunto de A que contém $\epsilon$ (string vazia)
e e fechado para operação de concatenação. $A^*$ também pode ser descrito como o conjunto
de todos os elementos que podem ser formados através da concatenação de zero ou mais elementos de $A$.

O fecho de Kleene pode ser definido recursivamente da seguinte
maneira:

\begin{center}
\begin{tabular}{lll}
$A_0     $ & = &  $\{ \epsilon \}$\\
$A_{i+1} $ & = &  $\{ wv : w \in A_{i}, v \in A\}$\\
\end{tabular}
\end{center}

Logo,

\begin{center}
$A^* = \bigcup_{i=0}^{\infty} A_i = A_0 \cup A_1 \cup \ldots$
\end{center}

Defina a função \texttt{kleene :: [a] -> [[a]]} tal que (\texttt{kleene xs}) devolve o fecho de Kleene do conjunto xs. Por exemplo,

\texttt{take 12 (kleene "01") == ["","0","1","00","01","10","11","000","001",
"100","101","010"]}

\texttt{take 20 (kleene "01") ==["","0","1","00","01","10","11","000","001",
"100","101","010","011","110","111","0000","0001","1000","1001","0100"]}

\item Considere o seguinte algoritmo que gera uma sequência de inteiros. Comece com um inteiro positivo n. Se
n é par, divida por 2. Se n é ímpar, multiplique por 3 e some 1. Repita o processo com o novo valor de n,
o processo termina quando n = 1. Por exemplo, comece com n = 6,

\texttt{[6, 3, 10, 5, 16, 8, 4 , 2, 1]}

Esta sequência é conhecida como sequência Collatz. Uma conjectura bastante conhecida defende
que este algoritmo de fato  termina para todo n positivo.

\begin{enumerate}

\item Defina a funcão \texttt{Collatz :: Integer -> [Integer]}

com o valor n usando a função \texttt{itera}. Por exemplo,

\texttt{collatz 13 = [13,40,20,10,5,16,8,4,2,1]}


\end{enumerate}
